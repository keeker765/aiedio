# AI Video Generation Pipeline - Team Roles & Boundaries

## 1. 团队职责分配方案 (5人编队) 与 目录映射

为了支撑从底层 AI 生成到前端展示的闭环，系统目录和团队职责划分如下：

### 角色 1：架构师与 AI 引擎核心 (对应目录: core_engine/, docs/)
- **职责**：项目整体架构设计、开发 AssetBuilder 策略工厂核心类、对接多模态大模型 API 与 LLM 推理调度。

### 角色 2：爬虫与数据工程师 (对应目录: crawler/)
- **职责**：多源高并发数据采集（Playwright/requests），构建清洗管道，为 LLM 的创意生成提供自动化的底层语料。

### 角色 3：后端业务开发工程师 (对应目录: ackend/)
- **职责**：服务器端开发（用户、权限、计费），管理海量 OSS 媒体素材库，提供标准化 Web API 以及任务队列通道。

### 角色 4：前端开发与核心交互工程师 (对应目录: rontend/)
- **职责**：实现主控台智能组件与复杂状态管理（Redux/Zustand），接入 WebSocket 监控长连接以及视频切片编排编辑器核心逻辑。

### 角色 5：前端开发与 UI/UX 设计师 (对应目录: rontend/)
- **职责**：高保真原型绘制，基础组件（Dumb Components）CSS 样式切图，以及产品动效和响应式终端适配。

## 2. 团队协作边界与单向数据流机制

为了防止内耗与 Git 代码冲突，制定以下护栏：

### 2.1 核心边界
- **AI 核心 vs 业务后端**：核心引擎(core_engine)不处理任何用户鉴权与积分扣除，永远只收发异步任务队列；ackend负责兜底。
- **爬虫数据 vs 后端持久化**：crawler不直连生产库，统一打“内部请求 API”让后端落盘归档。
- **上下游前端配合**：UI工程师专注纯“展示类/木偶组件”样式，交互工程师负责将其套入业务逻辑流中，坚决避免混编。

### 2.2 工作流闭环流转
1. **[Crawler]** -> 抓取热榜并清洗。
2. **[Backend]** -> 收到数据并存储在素材池。
3. **[Frontend Teams]** -> 呈现出给用户的热点UI控制台。
4. **[User]** -> 调整提交任务，触发后端网关。
5. **[Core Engine]** -> 消费任务，大模型联合生成后回传文件给后端。
6. **[Frontend]** -> 通过 WebSocket 收到完成通知大结局。
