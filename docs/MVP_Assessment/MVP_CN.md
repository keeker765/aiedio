## 1. MVP Scope Determination (5 pts)

**选择的2个 User Stories**：

| US   | 功能                                   | 理由             |
| ---- | -------------------------------------- | ---------------- |
| US-1 | Narrative Flow (故事板生成)            | AI自动创意的核心 |
| US-4 | One-Click Composition (爬虫+视频+合成) | 完整自动化闭环   |

**排除的理由**：US-2(人物一致-后续) / US-3(品牌控制-后续) / US-5(耗时优化)

---

## 2. User Story Complexity (3 pts)

### US-1: Narrative Flow — 🔴 High

- **多步顺序依赖**：4步LLM链（主题 → 大纲 → 场景 → 分镜），每步输出是下一步输入，无法并行
- **质量控制难**：创意输出主观，难以自动验证，需反复迭代Prompt
- **格式约束严格**：输出必须是结构化JSON，供下游视频生成消费
- **视频提示词对齐**：生成的场景描述必须精确到足以通过Video Gen API产出连贯视频片段——含糊或不一致的描述会导致无法使用的输出

### US-4: One-Click Composition — 🟡 Medium

- **模块可解耦**：5个子模块（爬虫 / 故事生成 / 视频API / 字幕 / 合成）接口清晰，可分工并行开发
- **成熟工具链**：依赖MoviePy/FFmpeg和Video Gen API，文档充足
- **复杂度集中在编排层**，而非算法设计

## 4. MVP Completeness (2 pts)

用户从**热点发现(爬虫) → AI创意(故事板) → 视频生成 → 自动合成**的闭环中，获得发布就绪的短视频。

**答案**：✅ 完整

| US   | 能删？                     |
| ---- | -------------------------- |
| US-1 | ❌ 无创意=无AI价值         |
| US-4 | ❌ 无爬虫和合成=无产品交付 |

**答案**：✅ 最小（2个都必需）

## 6. MVP Realism (2 pts)

**5人团队能完成吗？**

**时间规划**：

- Week 1：后端路由 + 前端框架 + 引擎Stub
- Week 2：US-1故事生成 + 爬虫基础
- Week 3：US-4视频生成、音乐、字幕、合成
- Week 4：测试、修复、集成

**工作量分配**：

- Wu Ke (核心引擎)：US-1 = 50小时
- Lu Yi (后端)：数据流、队列、存储 = 25小时
- Hu Yuxuan (爬虫)：US-4爬虫部分 = 15小时
- Liu/Li (前端)：UI界面 = 10小时

**总计**：~100小时 / 800小时可用 = 负载合理

**主要风险**：

- LLM多步推理质量 → 多套Prompt模板缓解
- Video Gen API限流 → 早期沟通配额 + Plan B

**答案**：✅ 可实现

## 7. KPI Definition

**KPI**：视频总播放量 ≥ 1万

- 10名用户生成50个视频，上传至平台，7天总播放数 ≥ 1万
- **为什么与收益相关**：高播放 → 用户持续使用 → 付费转化
- **验收**：50视频 × 200播放/个 = 1万

## 8. Implementation Tools (技术工具栈)

| 层级     | 工具                     | 用途                                 |

| 后端框架 | FastAPI (Python 3.10+)   | REST API、任务队列、数据流           |

| 版本控制 | Git + GitHub             | 协作开发、代码管理                   |
